## OSINT
- OSINT(Open Source Intelligence)とは何かを詳しく
- CTFでよく問われる具体的な対象（画像解析、位置特定、メタデータなど）が不足
- ツール名（Google検索、WHOIS、ExifToolなど）や具体的手順がない
- 何をすればいいかわかりづらい
- OSINTを解く流れが欲しい
- OCRについての説明不足
- 「OSINTの適切な利用」やプライバシー配慮の言及
- EXIF情報についての解説が不適切
- クロスチェック（複数ソース比較）、一次情報 vs 二次情報の区別、改ざん・フェイク検出（画像・動画）、発信者の意図やバイアス分析
- site:検索やGoogle検索演算子、画像の逆検索（Google Images / TinEye）、whois や nslookup、Wayback Machineで過去比較


## Crypto 
- 暗号化やBase64の解説不足
- 暗号化 → 復号（鍵あり）、ハッシュ → クラック（辞書攻撃 / レインボーテーブル）、判別方法（長さ・形式・アルゴリズム）
- ツール例なし、CyberChef、hashcat、手順なし
- シーザー暗号の解説不足
- ヴィジュネル暗号の解説不足
- どうやって書くか
- 解読の流れが不適切
- 解き方の方向性が欲しい
- Base64「暗号との違い（鍵の有無）」が弱い
- 具体例がない
- RSA暗号の解説不足
- 「mod」「逆数」などの説明がない
- 「鍵長が短い＝計算量が小さい」という本質（2^nの探索空間）が説明されていない
- 手法の名前はあるが具体例が弱い
- なぜ短いと危険か

## Forensics
- メモリダンプの目的
- 不足している代表例：ネットワーク接続情報、コマンド履歴、復号前データ（平文）、DLL / モジュール、レジストリ情報、認証情報（資格情報）
- 具体的な解析例がない
- フォレンジック初動として重要な観点（ハッシュ値・パーティション構造など）が抜けている
- ツールや具体例がない（例：fdisk, mmls, sha256sumなど）
- ステガノ特有の重要手法（LSB解析・strings・binwalk等）が不足
- 初動（file / strings / binwalk）、フォーマット確認、ステガノ専用ツール（zsteg / steghide）が抜けている
- コマンド例が一切ない
- ツールの使い方説明なし
- 「タイムスタンプの重要性」など基本は正しい,しかし重要な観点（タイムゾーン差・ログ間の時刻ズレ・改ざん検知など）が欠落
- 「整合性」などの用語説明なし
- 改ざんの可能性：ハッシュ値比較（改ざん検知の基本）
- タイムスタンプの矛盾（MAC timesなど）、ログ間の相関、メタデータの不整合、取得過程（チェーン・オブ・カストディ）
