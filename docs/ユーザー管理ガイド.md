# ユーザー管理ガイド

サーバー側でユーザー名やパスワードを変更する方法を説明します。

## 方法1: 管理者用APIを使用する（推奨）

管理者権限でログインした状態で、以下のAPIエンドポイントを使用できます。

### エンドポイント
```
PUT /admin/users/:userid/profile
```

### リクエスト例

#### ユーザー名のみ変更
```bash
curl -X PUT http://localhost:3000/admin/users/testuser/profile \
  -H "Content-Type: application/json" \
  -H "Cookie: connect.sid=<セッションID>" \
  -d '{"username": "新しいユーザー名"}'
```

#### パスワードのみ変更
```bash
curl -X PUT http://localhost:3000/admin/users/testuser/profile \
  -H "Content-Type: application/json" \
  -H "Cookie: connect.sid=<セッションID>" \
  -d '{"password": "newpassword123"}'
```

#### ユーザー名とパスワードの両方を変更
```bash
curl -X PUT http://localhost:3000/admin/users/testuser/profile \
  -H "Content-Type: application/json" \
  -H "Cookie: connect.sid=<セッションID>" \
  -d '{"username": "新しいユーザー名", "password": "newpassword123"}'
```

### JavaScript（fetch）での使用例
```javascript
// ユーザー名とパスワードを変更
fetch('/admin/users/testuser/profile', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json'
  },
  credentials: 'include', // セッションクッキーを送信
  body: JSON.stringify({
    username: '新しいユーザー名',
    password: 'newpassword123'
  })
})
.then(response => response.json())
.then(data => console.log(data));
```

## 方法2: スクリプトを使用する

コマンドラインから直接データベースを操作するスクリプトを使用できます。

### 使用方法

#### ユーザー名のみ変更
```bash
node scripts/update_user.js testuser --username "新しいユーザー名"
```

#### パスワードのみ変更
```bash
node scripts/update_user.js testuser --password "newpassword123"
```

#### ユーザー名とパスワードの両方を変更
```bash
node scripts/update_user.js testuser --username "新しいユーザー名" --password "newpassword123"
```

## 方法3: SQLiteコマンドラインで直接操作する

データベースファイルを直接操作する方法です。

### データベースファイルの場所
```
db/users.db
```

### SQLiteコマンドの使用例

#### データベースに接続
```bash
sqlite3 db/users.db
```

#### ユーザー名を変更
```sql
UPDATE users SET username = '新しいユーザー名' WHERE userid = 'testuser';
```

#### パスワードを変更（注意: ハッシュ化が必要）
パスワードはbcryptでハッシュ化されているため、直接SQLで変更する場合は事前にハッシュ化が必要です。

Node.jsでハッシュを生成:
```javascript
const bcrypt = require('bcrypt');
bcrypt.hash('newpassword123', 10).then(hash => console.log(hash));
```

生成されたハッシュを使用:
```sql
UPDATE users SET password = '$2b$10$生成されたハッシュ' WHERE userid = 'testuser';
```

### 現在のユーザー情報を確認
```sql
SELECT userid, username, role FROM users WHERE userid = 'testuser';
```

## 注意事項

1. **パスワードのハッシュ化**: パスワードは必ずbcryptでハッシュ化してから保存してください。平文で保存しないでください。

2. **バリデーション**: 
   - ユーザー名: 3-50文字、危険な文字（<, >, ', "）を含まない
   - パスワード: 8文字以上
   - ユーザーID: 英数字とアンダースコアのみ、3-20文字

3. **セキュリティ**: 管理者権限が必要な操作です。適切な権限管理を行ってください。

4. **バックアップ**: データベースを変更する前に、必ずバックアップを取ることを推奨します。

## トラブルシューティング

### ユーザーが見つからない
- ユーザーIDが正しいか確認してください
- データベースファイルが正しい場所にあるか確認してください

### パスワードが変更できない
- パスワードが8文字以上であることを確認してください
- スクリプトを使用する場合は、bcryptモジュールがインストールされているか確認してください

### 権限エラー
- 管理者権限でログインしているか確認してください
- セッションが有効か確認してください
