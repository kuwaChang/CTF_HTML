# AIチューター完全ローカルセットアップガイド

このガイドでは、**完全にローカルで動作する**AIチューターのセットアップ方法を説明します。OpenAI APIは使用しません。

## 概要

AIチューターは、Ollamaを使用して完全にローカルで動作します。インターネット接続やAPIキーは不要です。すべての処理がローカルマシン上で実行されます。

## 必要な環境

- Node.js (v16以上推奨)
- Ollama（ローカルLLM実行環境）

## セットアップ手順

### ステップ1: Ollamaのインストール

#### Windows
1. https://ollama.com/download からOllamaをダウンロード
2. インストーラーを実行してインストール
3. Ollamaが自動的に起動します（タスクバーにOllamaアイコンが表示されます）

#### Linux/Mac
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

インストール後、Ollamaが自動的に起動します。

### ステップ2: 必要なモデルのダウンロード

Ollamaが起動したら、以下のコマンドで必要なモデルをダウンロードします。

#### ターミナル（コマンドプロンプト）を開く

**Windows:**
- PowerShell または コマンドプロンプトを開く

**Linux/Mac:**
- ターミナルを開く

#### LLMモデル（チャット用）のダウンロード

```bash
# 推奨: llama3.2（軽量で高性能、約2GB）
ollama pull llama3.2
```

**その他の選択肢:**
```bash
# より軽量（約1GB、低リソース向け）
ollama pull llama3.2:1b

# より高性能（約5GB、高リソース向け）
ollama pull llama3.1:8b
```

#### 埋め込みモデル（ベクトル検索用）のダウンロード

```bash
# 推奨: nomic-embed-text（軽量で高性能、約137MB）
ollama pull nomic-embed-text
```

### ステップ3: モデルの確認

ダウンロードが完了したか確認：

```bash
ollama list
```

以下のように表示されれば成功です：

```
NAME                  ID           SIZE    MODIFIED
llama3.2              abc123...    2.0 GB  2 hours ago
nomic-embed-text       def456...    137 MB  1 hour ago
```

### ステップ4: プロジェクトの依存関係をインストール

CTFサーバーのプロジェクトディレクトリで以下を実行：

```bash
npm install
```

### ステップ5: サーバーの起動

環境変数の設定は**不要**です。デフォルトでローカルLLMが使用されます。

```bash
node server.js
```

初回起動時、以下のメッセージが表示されます：

```
✅ ローカルLLM（Ollama）を初期化しました: llama3.2 at http://localhost:11434
✅ ローカル埋め込みモデル（Ollama）を初期化しました: nomic-embed-text at http://localhost:11434
📚 知識ベースを読み込み中...
📝 XX個のチャンクを生成しました
🔄 埋め込みベクトルを生成中...
✅ 知識ベースのインデックス化が完了しました
```

### ステップ6: AIチューターにアクセス

ブラウザで以下のURLにアクセス：

```
http://localhost:3333/tutor
```

ステータスバーに「準備完了 (ローカルLLM: llama3.2, 知識ベース: XX件)」と表示されれば成功です！

## 使い方

### 基本的な使い方

1. **質問を入力**: チャット画面のテキストエリアに質問を入力
2. **送信**: 「送信」ボタンをクリック、またはEnterキーを押す
3. **回答を確認**: AIチューターからの回答と参考資料を確認

### クイックアクション

以下のボタンからよく使う質問を素早く送信できます：

- 💡 **解き方を聞く**: 問題の解き方について質問
- 🛠️ **ツールを聞く**: 使用すべきツールについて質問
- 📚 **基礎知識を聞く**: カテゴリーの基礎知識について質問

### 問題ページからの連携

問題ページからAIチューターにアクセスする場合、URLパラメータで問題情報を渡せます：

```
http://localhost:3333/tutor?category=Easy1&questionId=beginner1
```

この場合、AIチューターは現在取り組んでいる問題の情報を考慮して回答します。

## トラブルシューティング

### Ollamaが起動していない

**症状:** ステータスバーに「⚠️ Ollamaが起動していません」と表示される

**解決方法:**
1. **Windows**: タスクバーにOllamaアイコンがあるか確認。ない場合は、スタートメニューからOllamaを起動
2. **Linux/Mac**: ターミナルで `ollama serve` を実行
3. ブラウザで `http://localhost:11434` にアクセスして、Ollamaが応答するか確認

### モデルが見つからない

**症状:** 「モデルが見つかりません」というエラー

**解決方法:**
1. モデルがダウンロードされているか確認: `ollama list`
2. モデルをダウンロード:
   ```bash
   ollama pull llama3.2
   ollama pull nomic-embed-text
   ```

### 知識ベースの初期化に失敗する

**症状:** 「埋め込みモデルの初期化に失敗しました」というエラー

**解決方法:**
1. 埋め込みモデルがダウンロードされているか確認: `ollama list`
2. 埋め込みモデルをダウンロード: `ollama pull nomic-embed-text`
3. Ollamaが起動しているか確認: `ollama list` が正常に動作するか

### 応答が遅い

**原因:**
- モデルが大きい
- システムリソース（CPU/メモリ）が不足している
- 初回の応答生成は時間がかかる（モデルの読み込み）

**解決方法:**
1. より軽量なモデルを使用:
   ```bash
   ollama pull llama3.2:1b  # より軽量
   ```
2. システムリソースを確認（タスクマネージャーやhtopで確認）
3. 他のアプリケーションを閉じる
4. 初回応答後は、モデルがメモリに読み込まれているため、2回目以降は速くなります

### メモリ不足

**症状:** Ollamaがクラッシュする、または応答が生成されない

**解決方法:**
1. より軽量なモデルを使用:
   ```bash
   ollama pull llama3.2:1b
   ```
2. システムのメモリを増やす
3. 他のアプリケーションを閉じる

### ポート11434が使用中

**症状:** 「ポート11434が使用中」というエラー

**解決方法:**
1. 既にOllamaが起動している可能性があります。`ollama list` で確認
2. 別のポートを使用する場合は、環境変数を設定:
   ```bash
   export OLLAMA_BASE_URL=http://localhost:11435
   ```

## カスタマイズ

### 別のモデルを使用する

環境変数でモデルを変更できます：

```bash
# LLMモデルを変更
export OLLAMA_MODEL=llama3.1:8b

# 埋め込みモデルを変更
export OLLAMA_EMBEDDING_MODEL=mxbai-embed-large
```

### OllamaのベースURLを変更する

```bash
export OLLAMA_BASE_URL=http://localhost:11435
```

### .envファイルを使用する

プロジェクトルートに`.env`ファイルを作成：

```
OLLAMA_MODEL=llama3.2
OLLAMA_EMBEDDING_MODEL=nomic-embed-text
OLLAMA_BASE_URL=http://localhost:11434
```

`.env`ファイルを使用する場合は、`dotenv`パッケージをインストール：

```bash
npm install dotenv
```

`server.js`の先頭に以下を追加：

```javascript
require('dotenv').config();
```

## 推奨モデル

### LLMモデル（チャット用）

| モデル | サイズ | 推奨用途 | メモリ要件 |
|--------|--------|----------|-----------|
| `llama3.2:1b` | 1B | 軽量、高速、低リソース | 2GB RAM |
| `llama3.2` | 3B | **バランス型（推奨）** | 4GB RAM |
| `llama3.1:8b` | 8B | 高品質、高リソース | 8GB RAM |
| `mistral` | 7B | 高品質、多言語対応 | 8GB RAM |

### 埋め込みモデル（ベクトル検索用）

| モデル | サイズ | 推奨用途 | メモリ要件 |
|--------|--------|----------|-----------|
| `nomic-embed-text` | 137M | **軽量、高速（推奨）** | 1GB RAM |
| `mxbai-embed-large` | 335M | 高精度、高リソース | 2GB RAM |

## パフォーマンスの最適化

### GPUを使用する場合

Ollamaは自動的にGPUを検出して使用します。NVIDIA GPUを使用している場合、CUDAがインストールされている必要があります。

**確認方法:**
```bash
ollama show llama3.2
```

GPUが使用されている場合、出力にGPU情報が表示されます。

### CPUのみの場合

CPUのみで実行する場合、軽量なモデルを使用することを推奨します：
- LLM: `llama3.2:1b` または `llama3.2`
- 埋め込み: `nomic-embed-text`

## セキュリティ

- ローカルLLMは完全にローカルで実行されるため、データが外部に送信されることはありません
- Ollamaはデフォルトで`localhost:11434`でリッスンします（外部からはアクセス不可）
- 本番環境で使用する場合は、適切なファイアウォール設定を行ってください

## よくある質問

### Q: インターネット接続は必要ですか？

A: モデルのダウンロード時のみ必要です。ダウンロード後は完全にオフラインで動作します。

### Q: APIキーは必要ですか？

A: いいえ、完全にローカルで動作するため、APIキーは不要です。

### Q: どのくらいのメモリが必要ですか？

A: 推奨モデル（llama3.2 + nomic-embed-text）で約4-6GBのRAMが必要です。より軽量なモデル（llama3.2:1b）を使用すれば、2-3GBで動作します。

### Q: 応答速度はどのくらいですか？

A: 初回応答は数秒かかりますが、モデルがメモリに読み込まれた後は、1-3秒程度で応答が生成されます。CPUのみの場合、もう少し時間がかかる場合があります。

### Q: 複数のユーザーが同時に使用できますか？

A: はい、可能です。ただし、同時リクエストが多い場合、応答速度が遅くなる可能性があります。
