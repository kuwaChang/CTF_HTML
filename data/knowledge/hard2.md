# Hard2 問題別ヒント集

AI先生が Hard2 の質問に答えるときに参照するための補助資料です。  
直接の答えは示さず、着眼点と段階ヒントをまとめます。

---

## hard1（Trace-01: 破損したログインポータル）
- カテゴリー: web
- 着眼点:
  - 「表面ではなく裏側」はHTMLソースやコメント確認の合図
  - 見た目でなくDOM/ソース構造を調べる
- 段階ヒント:
  - 開発者ツールでページソースを確認する
  - コメントや非表示要素を探す
  - 不自然な文字列やパスを手掛かりに次の調査へ進む

---

## hard2（Trace-02: 傍受された通信記録）
- カテゴリー: web
- 着眼点:
  - pcap解析ではプロトコル・会話単位で絞るのが基本
  - 全パケットを生で読むよりフィルタ活用が重要
- 段階ヒント:
  - Wiresharkで通信全体を俯瞰する
  - HTTP/DNS/TCPストリームなどで候補を絞る
  - 会話再構成でメッセージや不自然なペイロードを確認する

---

## hard3（Trace-03: 不正なセッションデータ）
- カテゴリー: web
- 着眼点:
  - 問題文の「Cookie」に注目する設計
  - 値のエンコード形式（Base64/JWTなど）確認が有効
- 段階ヒント:
  - 開発者ツールの Application からCookieを確認する
  - 値をデコードし、可読情報がないか調べる
  - 更新・再読み込みで値の変化も比較する

---

## hard4（Trace-04: 偽装されたアクセス）
- カテゴリー: web
- 着眼点:
  - 他ユーザー情報の参照はIDOR（参照制御不備）が典型
  - まず公開されている経路情報（`robots.txt` など）を確認する
- 段階ヒント:
  - 指示どおりローカル実行して対象ページへアクセスする
  - `robots.txt` やリンクされていないパスを確認する
  - パラメータ変更で参照可能範囲の差を検証する

---

## hard5（Trace-05: サーバーの設定メモ）
- カテゴリー: web
- 着眼点:
  - パラメータ改変で別データに到達できるかを試す問題
  - `id` 系クエリは値変更テストの基本ポイント
- 段階ヒント:
  - 現在のURLパラメータを確認する
  - `id` の値を前後に変えてレスポンス差分を見る
  - 表示変化から隠し情報の存在を特定する
