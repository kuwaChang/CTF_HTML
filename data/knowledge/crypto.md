# 暗号学（Cryptography）基礎知識

## ROT13暗号
ROT13は、アルファベットを13文字ずらす単純な換字式暗号です。
- A→N, B→O, C→P... Z→M（逆も同じ）
- 暗号化と復号化が同じ操作なので、同じ手順で解ける
- FLAG{xxx}の形式で答えが隠されていることが多い

**問題のヒント:** 問題文に「SYNT{ebg13}」のように、FLAGに似た形の文字列が出たらROT13を疑う。FLAGとSYNTを1文字ずつ見比べると、ずれ方が分かり解読しやすい。答は必ずFLAG{...}の形式で提出する。

---

## Base64エンコード
Base64は、バイナリデータをテキストに変換するエンコーディング方式です。
- 末尾に`==`や`=`が付くことが特徴
- 英数字と`+`、`/`を使い、改行は`=`でパディングされる
- コマンド: `echo 'encoded_text' | base64 -d` または `base64 -d file.enc`（Linux/Mac）。WindowsではPowerShellの`[System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String("文字列"))`やオンラインデコーダーも使える
- デコードすると元のテキスト（平文やフラグ）が得られる

**問題のヒント:** 「鍵の石版」「標準エンコード」「写本室」などと書いてあり、英数字の羅列が与えられたらBase64を試す。特に末尾が`==`や`=`の文字列はBase64の典型。

---

## モールス信号
モールス信号は、点（.）と線（-）で文字を表現する通信方式です。
- 点と線（・と－、またはドットとダッシュ）の並びでアルファベット・数字を表す
- 数字も含む問題では、**数字に対応している**オンラインデコーダーを選ぶ必要がある
- SOSは `...---...` で表現される
- サーバーのステータスランプや「物理サーバー監視」「緊急停止コード」など、ランプの点滅で信号を送る設定の問題では、ランプの状態（点灯/消灯や長短）をモールスに変換してから解読する

**問題のヒント:** 「点と棒の暗号」「昔からある暗号」とあればモールス。復号サイトによっては数字に対応していないので、数字対応のサイトを使うよう促す。

---

## 座標暗号（A=1, B=2, …）
数字の羅列をアルファベットの位置に対応させる暗号です。
- A=1, B=2, C=3, …, Z=26 で変換する
- 例: 6-12-1-7 → F-L-A-G
- ハイフンで区切られた数字の列は、それぞれ1文字に対応していることが多い
- アンダースコア（_）は単語の区切り（スペース）を表すことがある

**問題のヒント:** 「ファイル管理のインデックス」「数字の羅列でファイルを指す」「独特の方式で管理」とあれば、各数字をA=1～Z=26に変換してつなげるとフラグになる。数字対応の座標デコーダーや変換表を使うとよい。

---

## 換字式暗号（単一換字・頻度分析）
一文字が別の一文字に固定で置き換わる暗号です。
- 英語では E, T, A, O, I, N が頻出するので、頻度の高い文字から推測できる
- 「単純な換字式暗号」「内部メモ」「チャットログを難読化」などの説明があれば、この方式を疑う
- ツール: https://quipqiup.com/ に暗号文を貼ると、頻度分析で解読を試みてくれる

**問題のヒント:** 文字の出現頻度を確認し、quipqiup.com で解読を試すよう案内する。答えはFLAG{...}形式なので、解読結果からその部分を抜き出す。

## Minecraft Enchantment Table Language（エンチャントテーブル言語）
Minecraftのエンチャントテーブルで使われる特殊な文字体系です。
- Unicodeの特殊文字で、通常のアルファベットとは見た目が違う
- ゲーム内のエンチャントテーブルと同じフォントで表示される
- 「趣味のゲームの文字」「若い研究員がパスワードに使った」などと書いてあれば、この方式を連想する

**問題のヒント:** 見慣れない記号の羅列（例: ⎓ꖎᔑ⊣{...}）が出たら、Minecraftのエンチャントテーブル用フォントの変換表や「Minecraft enchantment table decoder」で検索して解読する。答えはFLAG{...}で、中身は英語の単語（例: minecraft_enchant）のことが多い。

## Shakespeare Programming Language（SPL）・戯曲形式の隠しメッセージ
戯曲（演劇の台本）形式の英文で、プログラミング言語や隠しメッセージが絡む問題があります。
- **Shakespeare Programming Language（SPL）** は、シェイクスピアの戯曲風の見た目をした難解プログラミング言語
- 問題文に「The Infamous World Program」、登場人物に **Hamlet** や **Aemilia**、台詞に「Speak your mind!」などが出てきたらSPLを連想する
- 「ある特定の読み方で隠された3文字」とあれば、文脈から **SPL**（Shakespeare Programming Language の頭文字）を答える問題であることがある
- 戯曲が「プログラム」として説明されている場合も、SPLや、台詞の頭文字・特定の単語を拾う読み方を考える

**問題のヒント:** これはプログラムとして解釈する問題であること、登場人物名（Hamlet など）から有名なプログラミング言語を検索すると「Shakespeare Programming Language」が出てくることを伝える。隠された3文字はその略称（SPL）である場合がある。

## 解読の進め方（共通）
1. 問題文と配布ファイルから、使われていそうな方式（ROT13、Base64、モールス、座標、換字、特殊フォントなど）を推測する
2. フラグは多くの場合 `FLAG{...}` の形式なので、その形を手がかりに解読範囲を絞る
3. ツールは「ROT13 decoder」「Base64 decode」「Morse decoder」「Minecraft enchantment table decoder」「quipqiup」など、方式に合わせて案内する
4. 複合（例: モールスで出た結果をさらに2進数や別の方式で解く）が必要な問題もあるので、一段解いたあとにもう一度形式を確認する
